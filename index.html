Where I've worked.

Platform Engineer
Dreamlines GmbH · Cruise Industry 2022 — Present

Sole ops engineer — full ownership of cloud infrastructure, CI/CD, containers, identity, job orchestration, and AI tooling.

Infrastructure & Cost

  • Own the complete AWS estate — VPCs, ECS clusters, RDS, S3, IAM, CloudWatch, Cost Explorer. Every architectural decision, every incident.
  • Comfortable across the full AWS spectrum: from spinning up a basic EC2 instance to architecting production-grade platforms like self-hosted Airflow and Keycloak — always with security and cost built in from the start.
  • Ran a company-wide cost reduction programme: audited VPC flow logs, right-sized NAT gateways, restructured S3 access patterns, enforced resource tagging, and implemented S3 lifecycle policies to tier objects by access frequency. Migrated ~80% of production to EC2 Spot with ECS capacity providers and automatic On-Demand fallback.
  • Solid hands-on experience with AWS Bedrock — used for internal automations where GDPR constraints make routing data through external AI providers impractical.
  • Migrated container orchestration from EKS to ECS. Kubernetes was overkill — ECS simplified operations, reduced cost, and removed a maintenance surface nobody had bandwidth for.

Containers

  • Containerised legacy .NET Framework applications — multi-stage Docker builds, Windows/Linux compatibility, decade-old workloads running cleanly on ECS.
  • All public-facing services run as distroless, non-root containers. No shell, no package manager — minimal attack surface, limited blast radius.
  • Integrated Trivy into CI pipelines for image scanning — vulnerabilities caught before production, not after.
  • Reduced build times using Docker Buildx with layer caching, Alpine base images, multi-stage builds, and Docker Bake for complex multi-image definitions. containerd/runc familiarity, Podman for rootless workflows.

CI/CD

  • Migrated all CI/CD from Jenkins to Bitbucket Pipelines. Started with in-house EC2-based runners with auto-scaling (scale to zero at rest). Later moved to Bitbucket-hosted build minutes with custom pipeline images. Retained self-hosted EC2 runners only for Windows-based container builds — no hosted Windows builder exists in Bitbucket.

Internal Platforms on AWS

  • Designed and operate a shared Airflow platform used by several internal teams. UI and scheduler run as ECS services, DAGs synced from S3, EFS for persistent storage, logs to CloudWatch. 200+ production jobs run as individual ECS tasks via ECSRunTaskOperator — elastic scaling, no idle worker cost. Replaced AWS MWAA entirely. Owned the full Airflow 2.3 → 3.0 migration with no disruption.
  • Deployed self-hosted Keycloak on ECS as the company-wide identity provider — OIDC/OAuth2 SSO across all internal apps, user federation, client configs, role mappings. Subsequently migrated to Cloudflare Zero Trust, replacing the self-hosted layer with a managed, policy-driven access platform.
  • Deployed and automated an OpenVPN platform on ECS serving nearly 200 users across multiple countries. Zero-touch user provisioning and deprovisioning via Lambda, multi-AZ redundancy, full Terraform IaC. Built a Streamlit dashboard for ops to instantly provision users or kick active sessions without touching infrastructure.

AI & Automation

  • Familiar with RAG pipelines, vector databases, and OpenWebUI — tested and evaluated as part of the internal AI stack.
  • Used AWS Bedrock and Azure AI Foundry to deploy and run foundation models for internal automations. Both chosen for GDPR compliance — models run within the cloud environment, data never leaves.
  • Built a GDPR compliance gateway using Microsoft Presidio for PII detection and anonymisation, backed by a locally-hosted Ollama model. Any data heading to an external AI provider passes through it first.
  • Built an end-to-end call analytics pipeline: S3 ingestion → transcription → LLM-based KPI extraction → automated dashboards. Replaced a fully manual review process.
  • Built an invoice automation workflow using n8n: captures invoices from email, extracts details, pushes to relevant systems. Saved 1000+ manhours of manual processing.
  • Built several other workflow automations across teams — removing repetitive work and connecting systems that were never integrated.

Identity & Access

  • Experienced with Microsoft Entra — app registrations, API permissions, and access policy management.
  • Familiar with AWS Cognito — used for application authentication at the load balancer level, with users sourced from Azure AD federation and IAM.

Monitoring & Observability

  • Set up and maintain Grafana for infrastructure and application dashboards — collectors, data sources, and dashboards used daily by engineering and ops.
  • Configured Logstash for log ingestion and shipping across application and infrastructure streams.
  • Instrumented production ECS applications with OpenTelemetry (OTEL) — collector layer routing container metrics into CloudWatch without third-party tooling.
  • Built out CloudWatch from scratch: custom metrics from all applications and infrastructure, alarms, and dashboards covering the full stack.
  • Designed a priority-based on-call alerting system — production-only, tiered by severity. On-call gets paged for what matters, not drowned in noise.

Networking & Security

  • Manage Cloudflare as the company CDN and security perimeter: DNS, cache rules, WAF policies, DDoS protection, and performance configuration across all public properties.
  • Internal services DNS-resolved via Route53, only reachable through the VPN — no internal tooling exposed to the public internet.
  • Audited and cleaned up IAM — removed unused users, roles, and policies, tightened permission boundaries, enforced least-privilege across the board.
  • Strict zero-trust access model for developers and internal users — every role scoped to exactly what it needs, access granted deliberately, not by default.
  • Production logging and anomaly detection via CloudWatch — log groups, metric filters, and alarms configured to surface suspicious activity in real time.
  • MFA enforced for all IAM users including CLI access, using aws-mfa for temporary credentials — no long-lived access keys in the wild.
